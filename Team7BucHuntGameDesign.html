Mandatory Requirements:

  • Players are given a URL and an access code
  • This scavenger hunt works off a fixed list of popular locations on campus
  • Each location must have QR code (unique) to scan
  • These QR codes obtain a longitude and latitude coordinate pair
  • A server tracks player’s progress, tasks and data
  • Players must be able to see their progress
  • Browser-based solution
  • No cookies or user data stored between sessions
  • Players will have the ability to edit display name
  • Platform must be client-server
  • Internet connected device neutral
  • Player must receive a congratulatory email for completing all tasks
  • Validate previous implementation
  • User input for decoded QR
  • Admin dashboard

Optional Requirements:

  • Player has the ability to send a QR code from the interface. If a scanned QR code is invalid, it will result in an error.
  • Third-party QR application
  • User selection of text or email for sending QR and messages.
  • Users are able to change their UserName.
  • Integration of QR code scanning within player interface
